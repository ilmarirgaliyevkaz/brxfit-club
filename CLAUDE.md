## Что это

Рекламные лендинги фитнес-клуба **Bronx Fitness** (Астана, Казахстан).
Цель сайта одна — заявка в WhatsApp. Всё остальное вторично.

Общаемся по-русски.

## Жёсткие правила (не нарушать)

- **Оффер только один: 3 пробных занятия за 2990 ₸.** Не «первая тренировка бесплатно», не абонемент 29 990 ₸.
- **Никаких выдуманных данных.** Нет фактов — ставить `TODO(content):`, а не сочинять метрики, отзывы и кейсы.
- **Локация только ЭКСПО** — просп. Мангилик Ел 55/3, павильон B2.4. Не перечислять залы bronx.kz.
- **Метка источника — в начале WhatsApp-сообщения**, чтобы была видна в превью чата:
  `Заявка с сайта brxfit.club (Hyrox). Здравствуйте! ...`
- **Тренеры** — «мастера спорта и призёры турниров». Без «15 тренеров в 3 залах».
- **`BRONX J LO` удалён** — отсылка к реальному человеку, риск по правам. Не возвращать без прямого указания.

## Стек и бренд

- **Astro 4 + Tailwind 3 + TypeScript** (strict). Статика, без API.
- Токены: фон `#0A0A0A`, золото `#FFD700`, шрифт Inter.
- Классы-токены: `bg`, `surface`, `fg`, `muted`, `gold`, `line`. Хардкод цветов не использовать.

## Структура

```
src/
├── data/
│   ├── programs.ts    # каталог направлений; flagship + href + порядок витрины
│   └── site.ts        # GA4_ID, номер WhatsApp, общие константы
├── layouts/AdLandingLayout.astro   # meta, JSON-LD, gtag, трекинг кликов
├── components/sections/            # секции страниц
├── components/ui/                  # WhatsAppButton, ProgramCard
└── pages/                          # 5 страниц
```

- Витрина «С чего начинают» = `flagshipOrder` в `programs.ts`. Порядок задан вручную (Hyrox первый), **не по посещаемости**.
- Ссылка направления на свою страницу = поле `href` в `programs.ts`. `ProgramCard` сам отрисует `<a>` и стрелку.

## Ассеты

**Видео** (`public/video/`):
- `hyrox/hero.mp4`, `hyrox/overview.mp4` — промо Hyrox.
- `hyrox/reviews/1.mp4` (+ `.jpg`) — групповой отзыв; играет и на `/hyrox`, и в блоке отзывов главной.
- `reviews/1.mp4` (Ербол), `reviews/2.mp4` (+ постеры) — отзывы главной, 720×1280, со звуком, целиком.
- Промо fire / fight / teens в репозитории **пока нет** — витрина показывает постеры.
- Конвенция: промо направления — `video/<slug>/hero.mp4`; отзыв — `video/reviews/<n>.mp4`.
  К каждому ролику обязателен `.jpg`-постер: у `<video>` стоит `preload="none"`.
- Исходники (~1 ГБ) лежат вне репозитория, в `~/Downloads/MP3 for landing/`. В git не класть.
  Жать h264 до 3–8 МБ. Промо — без звука, короткий луп. Отзывы — звук и длительность сохранять.

**OG-картинки** (`public/`), 1200×630, единый шаблон:
- `og-image.png` (главная), `og-hyrox`, `og-fight`, `og-fire`, `og-teens`.
- Чёрный фон, золотая полоса слева, eyebrow `[ЧТО] · ASTANA`, крупный заголовок,
  белый подзаголовок, футер `BRONX FITNESS | ASTANA`. Серия на английском.
  Шрифт — Inter из `node_modules/@fontsource/inter` (woff2 → ttf).
- Эти же файлы служат постерами витрины (`posterBySlug` в `FlagshipPrograms.astro`)
  и остаются fallback, когда появится видео.
- «16 DIRECTIONS» на `og-image` привязано к реальному числу программ —
  при изменении каталога перегенерировать.

## Особенности окружения

- macOS, Node v26.
- **Умные/кудрявые кавычки ломают терминал.** В командах — только прямые кавычки или без кавычек.

## Смежное

- `bronx.kz` — отдельный сайт на hoster.kz. По закону РК домены `.kz` обязаны резолвиться на серверы физически в Казахстане, поэтому Vercel/Tilda для него не подходят. Этот проект (`brxfit.club`) — не `.kz`, ограничение не действует.
